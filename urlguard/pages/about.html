InControl

Block sneaky redirects and third-party background requests. You stay in control.

🛡 No data leaves your browser. No analytics. No accounts. Everything stays on your device.

What it catches

InControl watches two kinds of behavior that normal browsing tools ignore:

  • Redirect hijacks — you click a link to site A, it silently redirects through B, C, D before landing somewhere else. InControl surfaces the full chain.
  • Background fires — you load a page and it quietly fires requests to dozens of other domains (trackers, ad networks, fingerprinters). InControl lists them per tab so you can see exactly who your current page is talking to.

How to use the popup

Click the InControl icon in the toolbar. The popup has three tabs:

Activity

Live view of what's happening on the current tab. Redirect chains and third-party domains appear as a tree. Hover any domain to Block or Allow it.

Blocked

Every domain you've chosen to block. Future requests to these domains are stopped before they load. Hover to unblock.

Allowed

Domains you've marked as trusted. InControl stops flagging these in the Activity view. Useful for CDNs or services you rely on.

How blocking actually works

When you block a domain, InControl adds a rule to Chrome's declarativeNetRequest engine. From that point on, requests to that domain are refused at the network layer — the page never sees a response. Blocking persists across browser sessions.

Block this site

The Block this site button in the popup header blocks the domain of the page you're currently on. Visiting it again will fail — useful for shady redirect targets or sites you want to permanently avoid.

Block all 3rd party domains

When a page has many third-party requests, a Block all bar appears. One click blocks every third-party domain firing on that tab. Use carefully — legitimate sites often load from CDNs that you'll want to allow again afterward.

What shows up in "Activity"

A typical entry might look like:

  • A redirect chain: news-site.com → ads.net → tracker.io → destination.com
  • A background tree: the page domain at the root, with each third-party domain hanging off as a child node
  • A blocked badge: if InControl already stopped something on this tab, a red shield summarizes the count

Permissions, and why

InControl asks for broad permissions because redirect detection requires watching all tabs, not just one. Here is what each is used for:

  • webRequest + webNavigation — observe redirects and third-party fires in real time
  • tabs — read the active tab's URL to scope the popup view to the current page
  • storage — remember your blocked and allowed lists across sessions
  • declarativeNetRequest — enforce blocks at the network layer
  • <all_urls> — required to monitor behavior on any site you visit

⚠ Blocking breaks things. If a site stops working, check the Blocked tab — chances are a CDN or auth domain got swept in. Unblock and it'll work again.
